# Security Policy

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, email details of the vulnerability to: **tomjdraper1@gmail.com**

Include as much of the following as possible:

- Type of issue (e.g. data exposure, injection, authentication bypass)
- File paths and line numbers relevant to the issue
- Steps to reproduce
- Potential impact

You should receive a response within 48 hours. If the issue is confirmed, a fix will be prioritised and released as soon as possible.

## Scope

- The API Analytics server (`server/`)
- The dashboard (`dashboard/`)
- The middleware packages (`analytics/`)

## Supported Versions

Only the latest version of each middleware package is actively supported with security fixes.
